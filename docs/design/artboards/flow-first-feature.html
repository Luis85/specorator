branches we don't show inline

Where the flow forks

branch · b1
User cancels at frame 02

Esc closes the modal. No folder created. No state change. The cockpit stays empty. Cost of cancelling: zero.

branch · b2
Vault has no specs/ folder yet

/spec:start detects this and runs scaffold first (a one-time, reversible setup). User sees a "first time? we'll set you up" callout, not an error.

branch · b3
Slug collision

"magic-link-auth" already exists. We don't ask the user to rename — we suffix -2 and tell them in the post-state. Renamable later from the cockpit header.

branch · b4
User dismisses the coach

"I've got this" hides the coach for this step only. Cockpit stays — just quieter. Coach reappears at the next step unless globally disabled in settings.

branch · b5
User adds a ? question

Open question detected → it shows up in the cockpit's "open questions" tab. Hand-off at frame 06 would be blocked if any are unresolved.

branch · b6
User edits files outside Obsidian

Frontmatter is the source of truth (DR-004). Cockpit re-reads on file change. The user can hand-edit the stage marker if they want — we won't fight them.

where this flow could fail us

Risks & open questions

F1 · "Folder dump" feels intimidating

Frame 03 shows 12 files appearing in the vault. New users may panic. Mitigation: animate them in slowly · show a "we'll explain each as you go" line · only 00-vision opens automatically.

F2 · The one-liner is too short to derive a slug

"sign in" → sign-in? Too generic. Open question: do we ask for a name at all (orthodoxy), or quietly auto-suggest with a rename hint?

F3 · 60s is too short, or too long

We assume a fast first hand-off proves momentum. But step 1 (vision) may legitimately take 5 minutes of thinking. Test: watch real users, don't optimize for speed if quality suffers.

F4 · Coach says the same thing every time

If coach copy is canned, repeat users will tune it out. Mitigation: make coach copy a function of (stage, what's filled, what's missing) — never identical twice.
