flow · questions that live in the open

Open question — ? in a note → collected → resolved

Open questions are the honest residue of incomplete thinking. The plugin doesn't hide them — it surfaces them. A ? prefix in any spec file becomes a tracked question, visible in the cockpit, counted by the gate, and closable with a one-line answer that optionally becomes a decision note.

edge cases

When questions get complicated

A
wrong file
? in a non-spec file
The plugin only scans files inside a feature folder (files named with the step prefix 01- through 11-). A ? in a daily note or personal file is invisible to Specorator. No accidental captures.
only scans: features/*/0[1-9]-*.md
ignores: everything else
Scoped to spec files only. Personal notes stay personal.
B
stale question
? open for 14+ days
Questions older than 14 days turn amber in the cockpit panel and get a "stale" tag. The feature itself may also show as stale in the feature switcher. Not an error — a nudge.

14 days → amber. Not red. Still a nudge, not a block. Gate still counts it.
C
team mode
Peer resolves your ?
In team mode (shared vault via Obsidian Sync), a peer can change ? → ! in the spec file. On your next sync, the cockpit count drops. The resolved line carries the resolver's initials in a comment: <!-- @arch-lead -->

Async resolution works. Resolver credited in file. No server required.
design rationale for this flow

Why we made these choices

choice
? and ! — the whole syntax

One character to open a question. One character to close it. The answer lives on the same line. The file stays readable in any markdown editor. No plugin, no renderer, no schema. (DR-004)

choice
Gate blocks on open questions

An open question is explicit uncertainty. The Define→Build gate requires 0 open questions because you shouldn't build something you haven't decided yet. This is the one gate item that genuinely blocks — not a yellow flag. (DR-001)

choice
OQ → decision note is optional

Not every resolved question is a decision note. "Which colour for the button?" doesn't need an ADR. The coach nudges, the user judges. The two flows connect but neither requires the other.

choice
Cockpit panel collapses at zero

The OQ panel only appears when there are open questions. When the count hits zero, it disappears. Principle 5: quiet by default, loud only when something needs attention.

Open question OQ-2 (meta) · what if someone uses ? for bullet lists?

The ? prefix conflicts with any markdown author who uses question-mark bullets. Mitigations: (A) require ? with a space — rare as a list prefix. (B) scope scanning to a specific frontmatter tag (specorator: true). (C) use a different sigil (OQ:). Current design uses option A — simplest, lowest friction. Will revisit if false positives appear in beta.
