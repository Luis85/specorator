Settings — what the user can control

Specorator has no account, no server, no telemetry (DR-006). All configuration is local — stored in Obsidian's plugin data file. Settings are grouped into four areas: workflow, notifications, gates, and vault. This catalogue is the authoritative spec for the settings surface.

surface · obsidian settings panel

Where settings live

Specorator settings live in Obsidian's native Settings → Community plugins → Specorator panel. No custom settings window. This keeps the experience consistent with the rest of Obsidian and avoids building a second navigation system.

Settings → Community plugins → Specorator
Four sections below. All settings take effect immediately — no save button, no restart required.

Obsidian's native settings panel

Building a custom settings window would mean building a second navigation system. Obsidian's settings panel is already familiar, handles search, and keeps cognitive load low. The tradeoff: limited layout control. Worth it.

choice
Immutable settings marked "enforced"

DR-005 (snapshots) and DR-006 (no telemetry) are non-negotiable. Rather than hiding them from settings, they appear as read-only rows with an "enforced" or "always on" label. Users know they exist and why. Transparency over hiding.

choice
No save button — immediate effect

All settings take effect immediately on change. No "Apply" button, no restart required. This matches Obsidian's own convention and reduces the "did my change do anything?" anxiety.

Open: folder migration UX

If a user changes "Features folder" after features exist, all existing wikilinks break. The settings row warns about this, but the migration path (offer to rename the folder + update all links) is not yet designed. Deferred to post-MVP — the default folders are sensible and most users won't change them.
