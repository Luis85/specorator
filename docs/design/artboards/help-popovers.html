Hand-off
Finishing one of the 11 steps so the next one can begin.
When you hand off, Specorator marks the file as done, creates the next file from a template, and (optionally) notifies whoever owns the next step. You can undo within 24 hours.

The rules
Short, testable statements about how the feature must behave.
Each rule starts with "When… the system shall…". Specorator helps you write them so they're easy to check later. The technical name is EARS requirements.

"Decision note ADR" →
Decision note
A short doc that records why you made a big choice.
Like: "we picked magic-link over passwords because users have to be able to log in on shared computers." Future-you will thank present-you. The technical name is ADR.
▸ Start a decision note
"Open question" →
Open question
Something you wrote in a doc but aren't sure about.
Mark uncertainty with ? and a sentence. Specorator gathers them and won't let you hand off until they're resolved — so nothing gets glossed over.
▸ See open questions
"House rules constitution" →
House rules
Project-wide invariants every feature must respect.
"All user-facing strings translatable." "No new dependencies without a decision note." Hand-off checks include house rules automatically.

Coach
A small panel that tells you what to do next, in plain English.
Quiet by default — no cheerleading. Click I've got this to dismiss it for the current step. You can re-enable it from settings.
