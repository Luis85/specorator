Recovery — undo, revert, and "where am I?"

Three distinct situations, one shared principle: the user should never feel trapped. Undo covers the last 24 hours. Revert covers anything in the feature's history. "I'm lost" covers spatial disorientation — not knowing where in the 11 steps they are or how they got there.

Recovery corner cases

A
undo across gate
Undoing a gate cross
Gate crossings are undoable for 24h like any mutation. But the undo banner is more emphatic: it shows "This puts you back in the Define lane" with the lane colour shift visible in the confirm screen.
⚑ Undo puts you back in Define lane
Lane-change undo is explicit. Same mechanism, higher-stakes language.
B
vault deleted
File gone outside the plugin
If a user deletes a spec file from the OS file system, the cockpit shows a "file missing" warning on that step — not an error. Offers: restore from snapshot · skip this step · mark as intentionally removed.

Two devices, one vault
Obsidian Sync or iCloud can create conflicts. Specorator detects conflict copies of spec files and shows them in the feature timeline as a branch — pick one, discard the other. No auto-merge.

design rationale for this flow

Why we made these choices

choice
Snapshot before every mutation

Undo is only trustworthy if it's unconditional. DR-005 mandates a snapshot before every file write, gate cross, and hand-off. No exceptions, no "undo not available." (DR-005)

choice
Named moments, not timestamps

"After gate cross: Define → Build" is scannable in half a second. "2026-04-27T09:14:52Z" is not. The revert list derives snapshot names from lifecycle events — the same vocabulary the user already knows.

choice
Diff before revert

Revert R3 shows which files change before committing. This is the difference between a power tool and a dangerous tool. One extra screen; infinite trust gained.

choice
"Where am I" answered by the cockpit

Re-entry is the most common recovery scenario — not undo, not revert. The cockpit's "last activity" section is the first answer. The full timeline is the second. The feature switcher is the third. Three layers, progressively more detailed.

Open question · how many snapshots to keep?

The revert list shows all lifecycle snapshots (one per hand-off, gate, or major mutation). Features can have 11+ hand-offs, 2 gate crossings — so up to ~15 snapshots. Storage cost is low (markdown files only), but the list UX could get long. Option A: show all, grouped by lane (Define / Build / Ship). Option B: show only gate snapshots by default, expand for step-level detail. Leaning Option A for V1, revisit if lists exceed 20 items.
