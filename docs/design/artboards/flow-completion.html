Feature completion — ship, close, archive

Shipping is the only moment in the lifecycle that deserves a little weight. Not confetti — but a deliberate close. The feature moves from the active cockpit into an archive that's permanent, searchable, and useful for retrospectives. It can be re-opened if bugs surface.

Archive — the permanent record

Closed features live in archive/. They're read-only by default, searchable, and carry their full spec, gate history, decision notes, and retrospective. They can be re-opened — which is how post-ship bugs feed back into the system.

Edge cases & design rationale

A
abandoned feature
"I'm not building this anymore"
A feature can be abandoned at any step. Command palette: "Abandon feature." Requires a one-line reason. Moves to archive with status abandoned. Distinct from closed in the archive list — a different icon, a different colour.
✕ abandoned · step 4 · "pivoted away"
Abandoned ≠ deleted. Still in archive. Still searchable. Different icon.
B
retrospective required?
Can you skip the retrospective?
Step 11 (retrospective) can be skipped with a reason — same as any other step. But it's the only step that specifically blocks the close action without a skip reason. You can't close silently with a blank retrospective — that's the one enforced moment of reflection.
⚑ Step 11 is blank — add a note or skip with reason to close
Retro: the one enforced reflection. Skippable, but not silently. One line minimum.
C
house rules update
Retrospective → house rules
If the retrospective surfaces a pattern ("we always forget to add a strings section"), the coach offers: "Add this to your house rules?" One click opens the constitution file with the new rule pre-drafted. The learning loop closes here.
Add to house rules?
You've noted "strings section" twice. Make it a house rule?
not now
Retro → constitution. Learning loop. Patterns become rules. Rules become gate checks.
choice
Close is deliberate, not automatic

Completing step 11 doesn't auto-close. The user clicks "Close feature." This is intentional — closing is a commitment, and commitments deserve a moment of intentionality. The button is never hidden, but it requires a click.

choice
Archive is searchable, not a graveyard

Closed features are the most valuable design resource a team has — they show what worked, what took too long, what kept getting re-opened. The archive surface should feel like a library, not a trash bin. Cycle time, decision count, gate outcomes — all visible at a glance.
