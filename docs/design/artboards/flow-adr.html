design rationale for this flow

Why we made these choices

choice
"Decision note" not "ADR"

ADR is jargon that makes people feel like they're filing paperwork. "Decision note" is what it actually is: a note about a decision. Same file, same frontmatter, different first impression. (DR-003)

choice
Three fields, pre-filled

The blank page is the enemy of the decision note. Context, date, and feature name are auto-filled from where the user triggered the form. The title is inferred from the step content. The user only writes "why" and optionally "what else."

choice
Auto-link, not manual

The biggest source of decision-note abandonment is the linking step — people forget, lose context, or don't know where to put the wikilink. Saving the note places the link automatically. The file and the context are always paired. (DR-004)

choice
Nudge, not enforcement

The gate checks for a linked decision note, but it doesn't block on it the way it blocks on open questions. Three dismissals turns the check yellow, not red. The goal is habit, not compliance theatre. (Principle 5)

Open question · how much to pre-fill the title?

Frame 02 shows the title pre-filled ("Use Supabase magic links for auth"). This is the ideal but requires some NLP to extract a decision-shaped phrase from the step content. Fallback: pre-fill with the step name ("how it works") and let the user edit. Test plan: ship with step-name pre-fill first, see if users edit it into something better or just save the default.
