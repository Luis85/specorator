spec · 03

Plugin architecture & API

Obsidian plugin entry point, view types, event hooks, command registration, and the complete settings schema. Assumes familiarity with the Obsidian plugin API (obsidian.d.ts). Minimum Obsidian version: 1.4.0.

1Entry point

// main.ts — plugin entry point
export default class SpecoratorPlugin extends Plugin {
  settings: SpecoratorSettings;
  cache: SpecoratorCache;
  cockpitView: CockpitView | null;

  async onload() {
    await this.loadSettings();
    this.cache = new SpecoratorCache(this.app.vault, this.settings);
    await this.cache.build();          // scan vault on startup
    this.registerView(COCKPIT_VIEW_TYPE, (leaf) => new CockpitView(leaf, this));
    this.registerCommands();
    this.registerEventHooks();
    this.addSettingTab(new SpecoratorSettingTab(this.app, this));
  }

  async onunload() {
    this.cache.destroy();
  }
}

2View types

CockpitView

The primary interface. Registered as a leaf view via registerView(). Renders into a dedicated Obsidian leaf (right pane, or user-chosen layout). Not a modal.

COCKPIT_VIEW_TYPE = "specorator-cockpit"

class CockpitView extends ItemView {
  getViewType(): string { return COCKPIT_VIEW_TYPE; }
  getDisplayText(): string { return "Specorator"; }
  getIcon(): string { return "layout-dashboard"; }

  async onOpen() { this.render(); }

  render() {
    // Re-renders on:
    // - active leaf change (DR-002 · context inference)
    // - cache invalidation (file save)
    // - settings change
  }
}
Layout note: Cockpit A (three-pane) uses the left pane for the feature list, the centre pane for the active feature, and the right pane for context/OQ/coach. The plugin does not own Obsidian's left sidebar — it registers a leaf view and lets the user place it. The default open position is right.

GateView (modal-like)

Gate review is a full-screen overlay rendered inside the cockpit view — not an Obsidian Modal. This gives it full CSS control and allows the slow 400ms cross-fade.

3Event hooks

EventHandlerAction
vault.on('modify', file) onFileModify If file matches feature glob: invalidate cache for that feature, re-evaluate gate checks, update cockpit if visible.
vault.on('rename', file) onFileRename If old path matched feature glob: warn user that renaming step files breaks Specorator. Do not auto-migrate (post-MVP).
vault.on('delete', file) onFileDelete If deleted file is a step file: remove from cache, flag feature as having missing steps in cockpit.
workspace.on('active-leaf-change', leaf) onLeafChange Resolve active feature from leaf path (DR-002). Update cockpit context. No action if leaf is not a Markdown file in features/.
workspace.on('layout-change') onLayoutChange Re-render cockpit if its leaf is now visible.

5Settings schema

Stored in Obsidian plugin data (data.json). Loaded via this.loadData(). All settings take effect immediately — no restart, no save button.

interface SpecoratorSettings {
  // ── Vault layout (configurable; migration required if changed) ──
  featuresFolder:   string;   // default: "features"
  archiveFolder:    string;   // default: "archive"
  decisionsFolder:  string;   // default: "decisions"
  constitutionFile: string;   // default: "CONSTITUTION.md"

  // ── Workflow ──
  defaultFeatureSize: "ask" | "small" | "large"; // default: "ask"
  staleThresholdDays: number;   // default: 14
  retroRequired:      boolean;  // default: true
  snapshotRetention:  "all" | number;  // default: "all" (DR-005: always on)
  oqStaleThresholdDays: number; // default: 14

  // ── Notifications ──
  notificationLevel: "major" | "verbose"; // default: "major"
  coachNudges:       boolean;  // default: true
  decisionNotePrompt: boolean; // default: true
  reentryContext:    boolean;  // default: true
  reentryThresholdHours: number; // default: 24

  // ── Gates ──
  gateMode:          "solo" | "team"; // default: "solo"
  decisionNoteGateCheck: "hard" | "soft" | "off"; // default: "soft"
  gateScanTrigger:   "on-save" | "manual"; // default: "on-save"
  minEarsRules:      number;   // default: 4
  skipWarningThreshold: number; // default: 8

  // ── Team mode (gateMode = "team" only) ──
  committers:        string[]; // @handles, default: []
  minReviewers:      number;   // default: 1

  // ── Immutable (DR-005, DR-006) — shown in settings UI as read-only ──
  // snapshotBeforeEveryMutation: true  (always on, not stored)
  // noTelemetry: true                  (always on, not stored)
}

6Obsidian API surface used

APIUsed for
PluginBase class. Entry point, command registration, settings, event hooks.
ItemViewCockpit view. Renders in a leaf. Not a modal.
PluginSettingTabSettings panel in Obsidian Settings → Community plugins.
VaultFile read/write (read, modify, create, rename, delete). Also getMarkdownFiles() for vault scan.
MetadataCacheFrontmatter parsing via getFileCache(file).frontmatter. Faster than raw file reads for repeated access.
WorkspaceActive leaf change events. Opening leaves. Reveal/focus.
NoticeStatus bar messages (tier 1 notifications only). Duration: 3000ms for info, 0 (sticky) for errors.
ModalSkip reason prompt, abandon reason prompt, gate override reason prompt. Not used for gate review itself.
FuzzySuggestModalFeature switcher (fuzzy search by feature name).
TFile, TFolderFile/folder type guards throughout.
Do not use: app.fileManager.renameFile() for plugin-managed renames without updating all wikilinks and frontmatter references. Use vault.rename() + a manual reference update pass. Obsidian's link update only covers user-visible wikilinks, not frontmatter strings.
